!2 !-Execute-!
'''!- Execute -!'''!- executes any SQL statement. The statement is specified as the first fixture parameter. There are no additional rows required for this command. -!

!-You can use query parameters in the DB-specific syntax (eg. -!'''!- @paramname -!'''!- for SQLServer and MySQL, and -!'''!- :paramname -!'''!- for Oracle). Currently, all parameters are used as inputs, and there is no option to persist any statement outputs. -!

{{{
!-!-!3 to execute statements, use the 'execute' command

|Execute|Create table Test_DBFit(name varchar(50), luckyNumber int)|

|Execute|Insert into Test_DBFit values ('Obi Wan',80)|

|Set parameter|name|Darth Maul|

|Execute|Insert into Test_DBFit values (@name,10)|

|Query|Select * from Test_DBFit|
|Name|Lucky Number|
|Darth Maul|10|
|Obi Wan|80|

|Execute|Drop table Test_DBFit|
}}}

!3 Expecting Exceptions

It is also possible to trap and check exceptions resulting from the '''!- Execute -!''' command. This is done in the same manner as with the [[!-Execute Procedure-!][.DbFit.DbFitReference.CommandReference.ExecuteProcedureCommand]] command, by using '''!-Execute Expect Exception-!'''. Continuing the above example:

{{{
|Execute Expect Exception|Insert into Test_DBFit values ('Yoda Yoda Yoda Yoda Yoda Yoda Yoda Yoda Yoda Yoda Yoda Yoda ',10)|
}}}
As the value inserted is too large for the column an exception should be generated and trapped (and the row will go green). See the documentation for [[!-Execute Procedure-!][.DbFit.DbFitReference.CommandReference.ExecuteProcedureCommand]] for more details.
